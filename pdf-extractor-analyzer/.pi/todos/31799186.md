## Description
Add a new extraction mode called "prompt" that allows users to define extraction requirements through a provided custom prompt. This offers maximum flexibility for custom extraction scenarios.

## Implementation Details
- [ ] Add `PROMPT` to `ExtractionMode` enum in schemas.py
- [ ] Create prompt template system:
  - Accept user prompt as string parameter
  - System prompt instructs model to follow user prompt exactly
  - Allow optional output format specification (JSON schema, text, etc.)
- [ ] Add config option for custom prompts:
  - `custom_prompt: str | None = None`
  - `prompt_output_format: str = "json"` (or "text", "markdown")
- [ ] Update pipeline to accept and validate custom prompts
- [ ] Add prompt length validation (max tokens for model)
- [ ] Consider prompt templates for common use cases
- [ ] Add CLI support: `--mode prompt --prompt "your prompt here"`

## Files Affected
- `src/pdf_extractor_analyzer/schemas.py` - Add mode to enum
- `src/pdf_extractor_analyzer/config.py` - Add prompt config options
- `src/pdf_extractor_analyzer/analyzer.py` - Handle custom prompt mode
- `src/pdf_extractor_analyzer/pipeline.py` - Pass prompt to analyzer
- `src/pdf_extractor_analyzer/cli.py` - Add --prompt flag

## Example Usage
```python
# Code
result = extractor.extract(
    "invoice.pdf",
    mode=ExtractionMode.PROMPT,
    prompt="Extract all line items with quantity, description, and price"
)

# CLI
pdf-extractor document.pdf --mode prompt --prompt "Extract table of contents"
```
