## Refined Description
Implement async processing for multi-page PDFs with controlled concurrency and rate limiting, while preserving current sync behavior.

This task now includes **async single-document, streaming, and async batch APIs**.

## Decisions Confirmed
- Scope: **`extract_async()` + `extract_streaming()` + `extract_many_async()`**
- SDK target: **Migrate to new Replicate async SDK style (`AsyncReplicate`)**
- `max_concurrent_pages`: **per-document**
- Default failure behavior: **continue with partial results**
- `extract_async()` aggregated output: **preserve original page order**
- `extract_streaming()` emission order: **page order**
- CLI support: **add `--async` only** (final JSON shape unchanged)
- Rate limiting: **semaphore + requests/sec throttle**
- Existing sync batch path: **keep `extract_many()` unchanged**

## Implementation Plan

### 1) Migrate Replicate integration to async-capable SDK
- [ ] Upgrade dependency/config to a Replicate SDK version exposing `AsyncReplicate`
- [ ] Keep current sync extraction paths functional after migration
- [ ] Ensure token/auth flow remains compatible (`REPLICATE_API_TOKEN` and config token)

### 2) Analyzer async API
- [ ] Add `analyze_page_async()` in `analyzer.py`
- [ ] Add async retry path (`_run_with_retries_async`) with non-blocking backoff (`await asyncio.sleep`)
- [ ] Add async structured-repair path equivalent to sync behavior
- [ ] Reuse existing prompt-building, JSON extraction, and output normalization logic

### 3) Rate limiting + concurrency controls
- [ ] Add per-document async semaphore (`max_concurrent_pages`)
- [ ] Add requests-per-second throttling for async calls (in addition to semaphore)
- [ ] Expose new config options in `ExtractorConfig` for both controls

### 4) Pipeline async methods
- [ ] Add `extract_async()` in `pipeline.py`
  - validate input and cache behavior equivalent to sync path
  - analyze page tasks concurrently (bounded)
  - aggregate successful page outputs in original page order
  - keep partial-failure details in metadata
- [ ] Add `extract_streaming()` async generator
  - run page analysis concurrently
  - emit results in **page order** (buffer out-of-order completions)
- [ ] Add `extract_many_async()`
  - async batch processing path
  - preserve input order in returned batch results
  - keep sync `extract_many()` unchanged

### 5) Partial-failure contract
- [ ] Define and implement a consistent metadata contract for partial page failures (page number + error)
- [ ] Ensure failures do not abort by default in async paths
- [ ] Ensure structured mode merges successful page outputs only, then validates/repairs as needed

### 6) CLI integration
- [ ] Add `--async` flag in `cli.py`
- [ ] With `--async`, route single input to `extract_async()` and multi input to `extract_many_async()`
- [ ] Keep output JSON structure identical to sync mode
